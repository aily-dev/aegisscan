Step‑by‑step usage guide

Follow these steps to install dependencies, launch the interactive menu, and run deep or focused scans against your targets.

Quickstart · CLI based
  1. 1

    Clone & prepare the project

    Clone the repository to your system and ensure you have Python 3, Bash, and required security tools installed. AegisScan runs from the project root. 

    git clone <repo-url>
cd z-Tools   # or your cloned folder
  2. 2

    Make the launcher executable

    The `aegis.sh` script provides a hacker‑style interactive menu that generates and runs the appropriate Python scan script for you. 

    chmod +x aegis.sh
./aegis.sh
  3. 3

    Choose a scan mode

    From the main menu select a scan: deep scan (1), normal scan (2), or a focused module such as SQLi (3), XSS (4), port scanning (5), directory brute‑force (6), subdomains (7), and many more.

  4. 4

    Enter the target URL

    Provide the full target URL. If you omit the scheme, AegisScan will automatically prepend http:// and warn you, so you always know exactly what is being scanned.

  5. 5

    Review results & JSON report

    Scan output shows discovered URLs, forms, ports, directories, and a breakdown of vulnerabilities by severity and type. Detailed data is stored in an output directory such as scan_results_YYYYMMDD_HHMMSS with scan_summary.json for further analysis.

Direct Python deep scan

You can also run the deep scanner directly via the Python entrypoint defined in main.py. 

python3 main.py https://example.com \
  --output my_scan \
  --proxy http://127.0.0.1:8080
  • Deep crawling with async HTTP client and TLS verification
  • Summaries grouped by severity and vulnerability type
  • Ports, directories, and external tools all in one report

Getting started

A safe, repeatable way to run your first scans with clear outputs.

Run the interactive launcher

Use the hacker‑themed menu to choose a scan mode, enter a target URL, and let the launcher generate and run the correct Python runner. 

chmod +x aegis.sh
./aegis.sh

Run deep scan directly (Python)

For automation or CI/CD, call the Python entrypoint and keep results in a dedicated output folder. 

python3 main.py https://example.com \
  --output my_scan

Where are results stored?

Deep scans create an output directory like scan_results_YYYYMMDD_HHMMSS and write a consolidated scan_summary.json that’s easy to parse and report.

CLI & scan modes

Pick the right mode for the job: quick checks, focused retests, or full deep engagements.

Deep Scan vs Normal Scan

  • Deep Scan: maximum coverage, external tools enabled, deeper crawling.
  • Normal Scan: faster, reduced depth, external tools disabled.
  • Use focused modules (SQLi/XSS/…) when you already know where to look.

Proxy & TOR workflows

Route requests through your proxy for debugging or through TOR where allowed by scope. 

python3 main.py https://example.com --proxy http://127.0.0.1:8080
python3 main.py https://example.com --tor

Operational checklist
